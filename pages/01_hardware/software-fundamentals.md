                操作系统原理
                
                
                    
                        进程与线程
                        
                            
                                进程
                                
                                    程序的一次执行过程
                                    拥有独立的内存空间
                                    进程间相互隔离
                                    创建和切换开销较大
                                
                            
                            
                                线程
                                
                                    进程内的执行单元
                                    共享进程的内存空间
                                    线程间通信更高效
                                    创建和切换开销较小
                                
                            
                        
                        
# 进程状态转换
"""
创建态 -> 就绪态 -> 运行态 -> 阻塞态 -> 终止态
     ↗              ↘    ↗            ↘
   调度            I/O请求        I/O完成      正常/异常结束
"""
                    
                    
                    
                        内存管理
                        
# 内存分配方式
"""
连续分配:
- 单一连续分配: 内存分为系统区和用户区
- 固定分区分配: 预先划分固定大小的分区
- 动态分区分配: 根据进程大小动态划分

离散分配:
- 分页存储管理: 将内存和进程地址空间划分为固定大小的页
- 分段存储管理: 按逻辑关系划分段
- 段页式存储管理: 分段和分页相结合
"""
                        
                            内存管理是操作系统的核心功能之一，直接影响系统性能和资源利用率。

文件逻辑结构:

# 编译器工作流程
"""
1. 词法分析: 将源代码分解为记号(token)
2. 语法分析: 构建语法树，检查语法正确性
3. 语义分析: 检查语义正确性，进行类型检查
4. 中间代码生成: 生成与机器无关的中间表示
5. 代码优化: 优化中间代码
6. 目标代码生成: 生成机器代码
7. 目标代码优化: 进一步优化生成的机器代码
"""

# OSI七层网络模型
"""
应用层(第7层): 为应用程序提供网络服务
表示层(第6层): 数据格式转换、加密解密
会话层(第5层): 建立、管理和终止会话
传输层(第4层): 端到端的数据传输(TCP/UDP)
网络层(第3层): 路径选择和逻辑地址(IP)
数据链路层(第2层): 物理地址寻址、差错校验
物理层(第1层): 比特流传输
"""

# 关系数据库核心概念
"""
表(Table): 数据存储的基本单位
行(Row): 表中的一条记录
列(Column): 表中的一个字段
主键(Primary Key): 唯一标识表中每一行
外键(Foreign Key): 建立表间关系的字段
索引(Index): 提高查询速度的数据结构

SQL语言:
- DDL(数据定义语言): CREATE, ALTER, DROP
- DML(数据操作语言): INSERT, UPDATE, DELETE
- DQL(数据查询语言): SELECT
- DCL(数据控制语言): GRANT, REVOKE
"""
                    
                    
                    
                        事务与并发控制
                        
# 数据库事务ACID特性
"""
原子性(Atomicity): 事务要么全部成功，要么全部失败
一致性(Consistency): 事务执行前后数据保持一致状态
隔离性(Isolation): 多个事务并发执行时相互隔离
持久性(Durability): 事务提交后对数据库的修改是永久的

并发控制:
- 锁机制: 共享锁、排他锁
- 时间戳: 基于时间顺序控制并发
- 乐观控制: 假设冲突较少，提交时检查
- 多版本并发控制(MVCC): 保存数据的多个版本
"""
