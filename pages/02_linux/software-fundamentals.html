软件基础

软件基础是计算机系统的核心组成部分，包括操作系统、编译原理、数据结构与算法等关键领域，为应用程序的开发和运行提供支撑。

深入理解软件基础有助于架构师设计出高效、稳定、可维护的软件系统。

操作系统原理

进程与线程

进程

  • 程序的一次执行过程
  • 拥有独立的内存空间
  • 进程间相互隔离
  • 创建和切换开销较大

线程

  • 进程内的执行单元
  • 共享进程的内存空间
  • 线程间通信更高效
  • 创建和切换开销较小
# 进程状态转换
"""
创建态 -> 就绪态 -> 运行态 -> 阻塞态 -> 终止态
     ↗              ↘    ↗            ↘
   调度            I/O请求        I/O完成      正常/异常结束
"""

内存管理

# 内存分配方式
"""
连续分配:
- 单一连续分配: 内存分为系统区和用户区
- 固定分区分配: 预先划分固定大小的分区
- 动态分区分配: 根据进程大小动态划分

离散分配:
- 分页存储管理: 将内存和进程地址空间划分为固定大小的页
- 分段存储管理: 按逻辑关系划分段
- 段页式存储管理: 分段和分页相结合
"""

内存管理是操作系统的核心功能之一，直接影响系统性能和资源利用率。

"""

事务与并发控制

# 数据库事务ACID特性
"""
原子性(Atomicity): 事务要么全部成功，要么全部失败
一致性(Consistency): 事务执行前后数据保持一致状态
隔离性(Isolation): 多个事务并发执行时相互隔离
持久性(Durability): 事务提交后对数据库的修改是永久的

并发控制:
- 锁机制: 共享锁、排他锁
- 时间戳: 基于时间顺序控制并发
- 乐观控制: 假设冲突较少，提交时检查
- 多版本并发控制(MVCC): 保存数据的多个版本
"""
