渲染概述：从代码到画面

当你在浏览器中输入一个URL并按下回车，浏览器需要完成一系列复杂的步骤才能将网页内容呈现在你面前。这个过程称为"渲染流水线"(Rendering Pipeline)，主要分为五个阶段：

1

解析 (Parsing)

将HTML、CSS代码转换为浏览器可理解的结构（DOM和CSSOM）

2

构建渲染树 (Render Tree Construction)

结合DOM和CSSOM，创建包含视觉信息的渲染树

3

布局 (Layout)

计算渲染树中每个元素的位置和大小（回流/重排）

4

绘制 (Painting)

将元素的视觉属性（颜色、边框等）绘制到图层上（重绘）

5

合成 (Compositing)

将所有图层合并为一个图像并显示在屏幕上

关键概念

渲染过程是浏览器最核心的功能之一，直接影响网页性能。理解这个过程能帮助开发者编写更高性能的代码。渲染流水线的每个阶段都可能成为性能瓶颈，尤其是布局和绘制阶段，因为它们通常计算密集。

解析阶段：从文本到结构

解析是渲染过程的第一步，浏览器需要将接收到的HTML和CSS文本转换为结构化的数据模型，以便后续处理。这一阶段主要生成两种树结构：DOM和CSSOM。

DOM 解析

文档对象模型(DOM)是HTML文档的结构化表示，是浏览器操作网页内容的接口：

  • 解析器从顶部开始逐行读取HTML
  • 将标签转换为DOM节点
  • 构建嵌套的树状结构，反映HTML的层次关系
  • 遇到JavaScript可能会阻塞DOM解析 
<html>
  <body>
    <h1>标题</h1>
    <p>段落</p>
  </body>
</html>

CSSOM 解析

CSS对象模型(CSSOM)是CSS样式的结构化表示，包含了所有样式规则：

  • 解析CSS规则，计算每个选择器的优先级
  • 构建树状结构，体现样式的继承关系
  • 不会被JavaScript阻塞，会并行解析
  • 样式规则会按照 specificity 进行覆盖 
body { color: black; }
h1 { font-size: 24px; }
p { line-height: 1.5; }

关键概念

DOM和CSSOM是相互独立的数据结构。HTML解析会构建DOM树，CSS解析会构建CSSOM树。DOM树描述内容，CSSOM树描述如何显示这些内容。JavaScript可以修改DOM和CSSOM，这也是为什么脚本执行可能会阻塞渲染的原因。浏览器会推迟JavaScript执行和DOM构建，直到CSSOM准备就绪，这就是"渲染阻塞"。

渲染树构建：结合结构与样式

渲染树(Render Tree)是DOM树和CSSOM树的结合体，它包含了页面上所有可见元素及其样式信息。渲染树的构建是为了准备后续的布局和绘制步骤。

渲染树的特点

仅包含可见元素

渲染树只包含会显示在屏幕上的元素，像<head>标签、display:none的元素及其子元素会被排除。

样式信息已计算

每个节点都包含了最终计算后的样式(computed style)，解决了样式继承和冲突问题。

结构与DOM相似但不同

渲染树结构通常与DOM树相似，但可能因为某些元素(如伪元素)而有所不同。

构建过程

从DOM树的根节点开始遍历，为每个可见节点匹配CSSOM中的样式规则，创建渲染对象。

关键概念

渲染树构建过程中最复杂的部分是样式计算，浏览器需要为每个元素确定最终的样式。这涉及到处理继承、优先级(specificity)和层叠(cascade)规则。理解CSS优先级规则对于预测元素最终样式至关重要。此外，复杂的CSS选择器会增加样式计算的时间，影响渲染性能。

布局计算：确定元素位置与大小

布局(Layout)，也称为回流(Reflow)或重排(Layout)，是根据渲染树计算每个元素的几何信息（位置、大小、尺寸）的过程。这一步骤决定了元素在页面上的确切位置和占据的空间。

布局过程的特点

  • 从根节点开始

    布局是一个递归过程，从渲染树的根节点开始，遍历整个树。父元素的尺寸会影响子元素的尺寸和位置。

  • 计算盒模型

    计算每个元素的内容区、内边距(padding)、边框(border)和外边距(margin)，确定元素的总尺寸。

  • 相互依赖

    元素的布局相互影响，例如一个元素的宽度变化可能导致其他元素换行或重新定位。

  • 性能成本高

    布局是渲染流水线中最耗时的步骤之一，尤其是对于复杂页面，因为可能需要大量计算。

哪些操作会触发回流？

修改尺寸和位置
  • • width, height, margin, padding
  • • top, left, right, bottom
  • • border-width, border
  • • display, float, position
文档结构变化
  • • 添加/删除DOM元素
  • • 移动DOM元素
  • • 修改内容（尤其是文本）
  • • 伪类激活（如:hover）
浏览器窗口变化
  • • 窗口大小改变
  • • 页面滚动
  • • 缩放页面
读取布局属性
  • • offsetWidth, offsetHeight
  • • clientWidth, clientHeight
  • • getComputedStyle()
  • • getBoundingClientRect()

关键概念

回流是一个昂贵的操作，因为它可能需要重新计算整个页面或部分页面的布局。优化回流的关键是减少触发回流的次数和影响范围。可以通过批量修改DOM、使用documentFragment、避免频繁读取布局属性、使用CSS containment属性限制布局范围等方式来优化。现代浏览器会使用"脏检查"机制，将多个回流操作合并为一个，以提高性能。

绘制阶段：填充像素

绘制(Painting)，也称为重绘(Repaint)，是根据渲染树和布局信息，将元素的视觉属性（颜色、背景、边框、阴影等）绘制到屏幕上的过程。这一步骤将几何信息转换为实际的像素颜色。

绘制过程的特点

绘制操作

绘制包括多种操作，如：

  • • 绘制文本
  • • 绘制颜色和渐变
  • • 绘制图像和边框
  • • 绘制阴影和圆角
  • • 绘制透明度

绘制顺序

绘制按照特定顺序进行，以确保正确的视觉层次：

  • • 背景颜色
  • • 背景图像
  • • 边框
  • • 子代元素
  • • 轮廓和阴影

回流与重绘的关系

触发关系
  • 回流一定会导致重绘 - 元素位置或大小改变后，必须重新绘制
  • 重绘不一定需要回流 - 仅颜色等视觉属性变化时无需重新计算布局

关键概念

重绘的性能成本通常低于回流，但频繁的重绘仍然会影响性能。优化重绘的策略包括：减少使用昂贵的CSS属性（如box-shadow、gradient）、将动画和过渡效果应用到transform和opacity属性（这些属性可以由合成器线程处理，不触发重绘）、使用CSS containment属性限制绘制区域。浏览器会将绘制操作合并，并使用图层技术减少需要重绘的区域。

合成显示：合并图层并展示

合成(Compositing)是渲染流水线的最后一步，将页面的所有图层合并为一个图像，并发送到屏幕显示。现代浏览器使用图层技术来优化渲染性能，尤其是动画和交互效果。

图层与合成的优势

独立更新

不同图层可以独立更新，一个图层的变化不会影响其他图层，减少不必要的重绘和回流。

硬件加速

合成操作由GPU处理，效率远高于CPU，尤其适合处理动画和变换效果。

复杂场景优化

对于视频、3D变换等复杂场景，图层可以显著提升性能和流畅度。

触发合成层的属性与优化

创建合成层的情况
  • 3D变换 (transform: translateZ())
  • 视频、Canvas和WebGL内容
  • opacity小于1的元素
  • 带有will-change属性的元素
  • 重叠在合成层上的元素
合成层优化策略
  • 使用transform和opacity实现动画
  • 合理使用will-change，避免过度使用
  • 控制合成层数量，避免图层爆炸
  • 减少图层大小和绘制区域
  • 避免图层之间的频繁重叠

关键概念

合成是现代浏览器渲染性能优化的关键技术，通过GPU加速来处理图层合并和动画效果。然而，过多的合成层会消耗大量内存和GPU资源，导致"图层爆炸"问题。优化合成层的关键在于平衡，只对需要频繁动画或更新的元素创建合成层。will-change属性可以提示浏览器元素可能发生的变化，但应谨慎使用，避免不必要的性能开销。理解合成原理有助于创建流畅的动画和高性能的网页。

总结：网页渲染的完整流程

  1. 1

    解析阶段：浏览器将HTML解析为DOM树，将CSS解析为CSSOM树，JavaScript可能会在此阶段修改DOM或CSSOM

  2. 2

    渲染树构建：结合DOM和CSSOM，创建只包含可见元素及其计算样式的渲染树

  3. 3

    布局计算：计算每个元素的位置和大小，这是一个递归过程，父元素会影响子元素

  4. 4

    绘制阶段：根据布局信息，将元素的视觉属性绘制到对应的图层上

  5. 5

    合成显示：将所有图层合并，通过GPU加速渲染，并最终显示在屏幕上

网页渲染是一个复杂的流水线过程，每个阶段都可能影响最终性能。作为开发者，理解这些底层原理有助于编写更高性能的代码，优化关键渲染路径，减少不必要的回流和重绘，合理利用合成层技术，从而创建流畅、响应迅速的用户体验。
