## 一、vLLM 简介

### 1.1 什么是 vLLM

vLLM 是一个高性能的大语言模型推理引擎，核心特点是 PagedAttention 技术：

- 将 GPU 显存分块管理，像操作系统分页一样高效
- 显存利用率提升 2-4 倍
- 支持 KV Cache 复用
- 吞吐量为 HuggingFace Transformers 的 24 倍

### 1.2 核心优势

| 特性 | 说明 |
|------|------|
| 高吞吐量 | PagedAttention 实现显存高效管理 |
| 连续批处理 | 动态批处理请求，减少空闲 |
| 张量并行 | 支持多GPU分布式推理 |
| 流式输出 | 支持 token 级流式返回 |
| OpenAI 兼容 | API 兼容 OpenAI 接口 |

## 二、安装部署

### 2.1 环境要求

```
硬件要求：
- NVIDIA GPU（至少 16GB 显存）
- CUDA 11.8 或 12.1
- 驱动版本 >= 450.80.02

软件要求：
- Python 3.8 - 3.11
- PyTorch 2.0+
```

### 2.2 安装命令

```bash
# 推荐：使用 pip 安装（CUDA 12.1）
pip install vllm

# 或从源码安装（获取最新特性）
git clone https://github.com/vllm-project/vllm.git
cd vllm
pip install -e .
```

### 2.3 快速启动

```python
from vllm import LLM, SamplingParams

# 初始化模型
llm = LLM(model='Qwen/Qwen2.5-7B-Instruct')

# 设置采样参数
sampling_params = SamplingParams(
    temperature=0.7,
    top_p=0.95,
    max_tokens=512
)

# 推理
outputs = llm.generate(["你好，请介绍一下自己", "什么是人工智能？"], sampling_params)

for output in outputs:
    print(output.outputs[0].text)
```

### 2.4 命令行启动服务

```bash
# 启动 OpenAI 兼容 API 服务
vllm serve Qwen/Qwen2.5-7B-Instruct \
    --host 0.0.0.0 \
    --port 8000 \
    --tensor-parallel-size 2 \
    --gpu-memory-utilization 0.9

# 访问 API
curl http://localhost:8000/v1/chat/completions \
    -H "Content-Type: application/json" \
    -d '{
        "model": "Qwen/Qwen2.5-7B-Instruct",
        "messages": [{"role": "user", "content": "Hello!"}]
    }'
```

## 三、核心配置参数

### 3.1 服务参数

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| --host | 服务监听地址 | 0.0.0.0 |
| --port | 服务端口 | 8000 |
| --tensor-parallel-size | 张量并行数（GPU数量） | 1-8 |
| --gpu-memory-utilization | GPU显存使用率 | 0.9 |
| --max-model-len | 最大序列长度 | 8192 |
| --max-num-seqs | 最大并发序列数 | 根据显存调 |
| --block-size | KV缓存块大小 | 16 |
| --swap-space | 交换空间大小(GB) | 4 |

### 3.2 采样参数

```python
sampling_params = SamplingParams(
    prompt_logprobs=None,
    logprobs=None,
    temperature=0.7,
    top_p=0.95,
    top_k=50,
    max_tokens=512,
    stop=None,
    repetition_penalty=1.0,
    length_penalty=1.0,
    early_stopping=False,
)
```

## 四、生产级部署

### 4.1 Docker 部署

```dockerfile
FROM nvidia/cuda:12.1.0-devel-ubuntu22.04

RUN apt-get update && apt-get install -y python3.11 python3-pip
RUN pip3 install vllm

WORKDIR /app

CMD ["vllm", "serve", "/models/Qwen2.5-7B-Instruct"]
```

### 4.2 Docker Compose

```yaml
version: '3.8'
services:
  vllm:
    image: vllm/vllm-openai:latest
    container_name: vllm-server
    ports:
      - "8000:8000"
    volumes:
      - /data/models:/models
    environment:
      - CUDA_VISIBLE_DEVICES=0,1,2,3
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 4
              capabilities: [gpu]
    command: >
      --model /models/Qwen2.5-72B-Instruct
      --tensor-parallel-size 4
      --gpu-memory-utilization 0.9
```

## 七、常见问题

### 7.1 显存不足 OOM
```bash
# 降低显存使用
--gpu-memory-utilization 0.8
# 减小最大长度
--max-model-len 4096
# 使用量化
--quantization fp8
```

### 7.2 推理速度慢
- 检查是否使用了正确的精度（BF16 vs FP16）
- 调整并发参数
- 考虑使用更小的模型
