## 二、必需开源软件清单

### 2.1 按层次分类

| 层次 | 软件 | 用途 | 优先级 | 备注 |
|------|------|------|--------|------|
| **推理服务** | vLLM / Ollama | LLM 推理 | 必须 | 二选一 |
| **向量数据库** | Milvus / Qdrant | 向量存储检索 | 必须 | 二选一 |
| **应用框架** | LangChain / LlamaIndex | RAG 开发 | 必须 | 二选一 |
| **API 服务** | FastAPI | 后端服务 | 必须 | 可替换 |
| **前端** | Vue3 / React | 用户界面 | 必须 | 二选一 |
| **缓存** | Redis | 会话缓存 | 推荐 | 可选 |
| **对象存储** | MinIO | 文件存储 | 推荐 | S3 兼容 |
| **消息队列** | Kafka / RabbitMQ | 异步任务 | 选配 | 大规模用 |
| **监控** | Prometheus + Grafana | 监控告警 | 推荐 | 可选 |
| **容器编排** | Docker Compose / K3s | 服务编排 | 必须 | 根据规模 |

### 2.2 按场景推荐

**场景一：最小可用部署（单台服务器）**

| 软件 | 版本 | 资源配置 |
|------|------|---------|
| Ollama | latest | 1 GPU, 16GB+ |
| Qdrant | 1.7+ | 8GB RAM |
| PostgreSQL | 15+ | 4GB RAM |
| Redis | 7+ | 2GB RAM |
| FastAPI | latest | 2CPU |
| Vue3 | 3.4+ | 前端独立 |
| Docker Compose | 2.0+ | - |

**场景二：生产级部署（多台服务器）**

| 软件 | 版本 | 资源配置 |
|------|------|---------|
| vLLM | latest | 4 GPU, 64GB+ |
| Milvus | 2.3+ | 集群模式 |
| PostgreSQL | 15+ | 主从复制 |
| Redis Cluster | 7+ | 高可用 |
| Kafka | 3.6+ | 3节点 |
| MinIO | latest | 分布式 |
| Kubernetes (K3s) | 1.28+ | - |
| Prometheus | 2.0+ | 监控集群 |

## 三、部署前准备

### 3.1 硬件要求

```
最低配置（单服务 POC）：
- CPU: 8 核
- 内存: 32GB
- 硬盘: 500GB SSD
- GPU: NVIDIA 3090 (24GB) 或同等

标准配置（生产）：
- CPU: 32 核
- 内存: 128GB
- 硬盘: 2TB SSD
- GPU: NVIDIA A100 (40GB) x 2
- 网络: 千兆内网
```

### 3.2 系统环境

```bash
# 操作系统要求
- Ubuntu 22.04 LTS (推荐)
- 或 CentOS 8 / Rocky 9
- 或 Windows Server 2019+

# 基础软件
- Docker 24.0+
- Docker Compose 2.20+
- NVIDIA Driver 535+
- CUDA 12.1+
- NVIDIA Container Toolkit
```

### 3.3 网络配置

```bash
# 内网环境需要配置
# 1. 镜像仓库（离线部署需要）
# - Harbor 私有镜像仓库
# - 或配置离线导入

# 2. DNS 配置
# - 内部 DNS 解析
# - 或 /etc/hosts 手动配置

# 3. 防火墙
# - 开放必要端口
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --permanent --add-port=443/tcp
firewall-cmd --permanent --add-port=8000/tcp
firewall-cmd --reload
```

## 六、快速部署脚本

### 6.1 一键部署脚本

```bash
#!/bin/bash
# deploy-all.sh - 一键部署基础服务

set -e

echo "=== 开始部署大模型应用平台 ==="

# 1. 创建目录
mkdir -p /data/{models,logs,postgres,redis,milvus,minio}
mkdir -p /data/models/{llm,embedding}

# 2. 创建网络
docker network create llm-net

# 3. 启动 PostgreSQL
docker run -d \
  --name postgres \
  --network llm-net \
  -e POSTGRES_PASSWORD=your_password \
  -e POSTGRES_DB=llm_app \
  -v /data/postgres:/var/lib/postgresql/data \
  -p 5432:5432 \
  postgres:15

# 4. 启动 Redis
docker run -d \
  --name redis \
  --network llm-net \
  -v /data/redis:/data \
  -p 6379:6379 \
  redis:7-alpine

# 5. 启动 Qdrant
docker run -d \
  --name qdrant \
  --network llm-net \
  -v /data/qdrant:/qdrant/storage \
  -p 6333:6333 \
  -p 6334:6334 \
  qdrant/qdrant

# 6. 启动 vLLM
docker run -d \
  --name vllm \
  --network llm-net \
  --gpus all \
  -v /data/models:/models \
  -p 8000:8000 \
  -e CUDA_VISIBLE_DEVICES=0 \
  vllm/vllm-openai:latest \
  --model /models/llm/Qwen2.5-7B-Instruct \
  --host 0.0.0.0 \
  --port 8000

# 7. 启动 FastAPI
docker run -d \
  --name fastapi \
  --network llm-net \
  -v $(pwd)/app:/app \
  -p 8080:8080 \
  -e DATABASE_URL=postgresql://postgres:your_password@postgres:5432/llm_app \
  -e REDIS_URL=redis://redis:6379 \
  -e QDRANT_URL=http://qdrant:6333 \
  -e VLLM_URL=http://vllm:8000 \
  your-fastapi-image:latest

# 8. 启动前端
docker run -d \
  --name frontend \
  --network llm-net \
  -v $(pwd)/dist:/usr/share/nginx/html \
  -p 80:80 \
  nginx:alpine

echo "=== 部署完成 ==="
echo "前端地址: http://your-server:80"
echo "API地址: http://your-server:8080"
echo "向量库: http://your-server:6333"
echo "LLM服务: http://your-server:8000"
```

### 6.2 验证部署

```bash
# 验证 LLM 服务
curl http://localhost:8000/v1/models

# 验证向量库
curl http://localhost:6333/collections

# 验证 API
curl http://localhost:8080/health

# 查看日志
docker-compose logs -f
```

## 七、离线环境配置

### 7.1 Harbor 镜像仓库

```yaml
# docker-compose-harbor.yml
version: '3.8'
services:
  harbor:
    image: goharbor/harbor-core:v2.9.0
    container_name: harbor
    restart: always
    privileged: true
    volumes:
      - /data/harbor/data:/var/lib/harbor
      - ./harbor.yml:/etc/harbor/harbor.yml
    ports:
      - "80:80"
      - "443:443"
      - "4443:4443"
```

### 7.2 镜像导出导入

```bash
# 在有网环境导出镜像
docker save -o images.tar \
  vllm/vllm-openai:latest \
  qdrant/qdrant:latest \
  postgres:15 \
  redis:7-alpine \
  nginx:alpine

# 传输到内网
scp images.tar user@内网服务器:/data/

# 内网导入镜像
docker load -i images.tar
```

## 八、安全配置

### 8.1 防火墙规则

```bash
# 只开放必要端口
# LLM 服务（内部）
firewall-cmd --permanent --add-port=8000/tcp

# API 服务
firewall-cmd --permanent --add-port=8080/tcp

# 前端
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --permanent --add-port=443/tcp

# 向量库（可选，内部使用）
firewall-cmd --permanent --add-port=6333/tcp

# 重载
firewall-cmd --reload
```

### 8.2 认证配置

```yaml
# Nginx 基础认证
# 1. 生成密码文件
htpasswd -bc /data/nginx/.htpasswd admin your_password

# 2. Nginx 配置
server {
    listen 443 ssl;
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    location / {
        auth_basic "Restricted";
        auth_basic_user_file /data/nginx/.htpasswd;
        proxy_pass http://backend;
    }
}
```
