## 一、微调 vs 蒸馏 vs RAG

### 1.1 概念对比

| 技术 | 原理 | 适用场景 | 资源需求 | 训练时间 |
|------|------|---------|---------|---------|
| **RAG** | 检索增强生成 | 知识问答、实时信息 | 低 | 无需训练 |
| **Prompt Tuning** | 调整提示词 | 任务适配 | 极低 | 分钟级 |
| **LoRA/QLoRA** | 低秩适配 | 领域适配 | 中等 | 小时级 |
| **全量微调** | 全部参数更新 | 深度定制 | 高 | 天级 |
| **模型蒸馏** | 大模型教小模型 | 模型压缩 | 高 | 小时~天级 |

### 1.2 如何选择

```
决策树：

需要更新知识？
├── 是 → RAG（实时知识）
│        或 知识微调（频繁更新知识）
└── 否 → 需要定制能力？
         ├── 是 → 风格/领域适配
         │        └── LoRA 微调
         └── 否 → 追求更低成本？
                  ├── 是 → 模型蒸馏
                  └── 否 → 直接使用基础模型
```

### 1.3 我们的选择

```
企业业务场景推荐：

1. 首选：RAG + 基础模型
   - 90% 场景适用
   - 成本低、效果好

2. 进阶：RAG + LoRA 微调
   - 需要特定风格
   - 需要复杂推理能力

3. 高阶：模型蒸馏 + 微调
   - 需要部署到边缘设备
   - 需要降低推理成本
```

## 四、LoRA 微调详解（重点）

### 4.1 LoRA 原理

```
LoRA = Low-Rank Adaptation

核心思想：冻结预训练模型参数，只更新少量额外参数

原始权重: W ∈ R(d×k)
LoRA 更新: W' = W + ΔW = W + BA
其中: B ∈ R(d×r), A ∈ R(r×k), r << min(d,k)

参数量对比：
- 全量微调 7B 模型: 70亿参数
- LoRA 微调 7B 模型: ~800万参数（减少99%）
```

### 4.2 LLaMA-Factory 可视化微调

```bash
# 启动 WebUI
llamafactory-cli webui

# 或命令行训练
llamafactory-cli train examples/train_full/qwen2_full.yaml
```

### 4.3 LoRA 训练配置

```yaml
# lora_config.yaml
model:
  model_name_or_path: Qwen/Qwen2.5-7B-Instruct
  precision: bf16  # 或 float16, 4bit

  template: qwen
  max_length: 2048

    - q_proj
    - k_proj

    - o_proj
    - gate_proj

    - down_proj

  gradient_accumulation_steps: 16
  learning_rate: 1.0e-4
  num_train_epochs: 3
  lr_scheduler_type: cosine
```

### 5.2 数据清洗

```python
from datasets import load_dataset
import json

def clean_data(examples):
    cleaned = []
    for ex in examples["messages"]:
        # 过滤空消息
        messages = [m for m in ex if m.get("content")]
        # 过滤超长对话
        total_len = sum(len(m["content"]) for m in messages)
        if total_len < 10000:
            cleaned.append({"messages": messages})
    return {"messages": cleaned}

# 加载数据集
dataset = load_dataset("json", data_files="raw_data.json")
dataset = dataset.map(clean_data, batched=True)
dataset.save_to_disk("cleaned_data")
```

### 5.3 数据增强

```python
# 同义词替换
import random

def augment_text(text):
    synonyms = {
        "上涨": ["走高", "上行", "升高"],
        "下跌": ["回落", "下行", "下降"],
        "分析": ["研判", "解读", "评估"]
    }
    
    for word, syns in synonyms.items():
        if word in text:
            text = text.replace(word, random.choice(syns))
    return text

def augment_data(examples):
    augmented = []
    for ex in examples["messages"]:
        aug_messages = []
        for m in ex:
            aug_messages.append({
                "role": m["role"],
                "content": augment_text(m["content"])
            })
        augmented.append({"messages": aug_messages})
    return {"messages": augmented}
```

### 6.5 Step 4: 训练代码

```python
# train_lora.py
import torch
from transformers import (
    AutoModelForCausalLM,
    AutoTokenizer,
    TrainingArguments,
    Trainer,
    DataCollatorForSeq2Seq
)
from peft import LoraConfig, get_peft_model, TaskType

def train():
    # 加载模型
    model = AutoModelForCausalLM.from_pretrained(
        args.model_name_or_path,
        device_map="auto",
        torch_dtype=torch.bfloat16
    )
    
    # 配置 LoRA
    lora_config = LoraConfig(
        task_type=TaskType.CAUSAL_LM,
        r=args.lora_rank,
        lora_alpha=args.lora_alpha,
        lora_dropout=args.lora_dropout,
        target_modules=args.target_modules,
        bias="none"
    )
    
    # 应用 LoRA
    model = get_peft_model(model, lora_config)
    model.print_trainable_parameters()
    
    # 加载数据
    from datasets import load_dataset
    
    def preprocess_function(examples):
        tokenizer.pad_token = tokenizer.eos_token
        result = tokenizer(
            [e["content"] for m in examples["messages"] for e in m],
            truncation=True,
            max_length=2048
        )
        return result
    
    dataset = load_dataset("json", data_files=args.data_path)
    dataset = dataset.map(preprocess_function, batched=True)
    
    # 训练参数
    training_args = TrainingArguments(
        output_dir=args.output_dir,
        per_device_train_batch_size=args.batch_size,
        gradient_accumulation_steps=4,
        learning_rate=args.learning_rate,
        num_train_epochs=args.num_train_epochs,
        bf16=args.bf16,
        logging_steps=args.logging_steps,
        save_steps=args.save_steps,
        warmup_steps=args.warmup_steps,
        report_to="wandb"
    )
    
    # 开始训练
    trainer = Trainer(
        model=model,
        args=training_args,
        train_dataset=dataset["train"],
        data_collator=DataCollatorForSeq2Seq(tokenizer, model=model)
    )
    
    trainer.train()
    model.save_pretrained(args.output_dir)

if __name__ == "__main__":
    train()
```

### 6.6 Step 5: 模型合并与导出

```python
# merge_lora.py
from peft import PeftModel
from transformers import AutoModelForCausalLM, AutoTokenizer
import torch

def merge_model():
    base_model = AutoModelForCausalLM.from_pretrained(
        "/data/models/Qwen2.5-7B-Instruct",
        device_map="auto",
        torch_dtype=torch.bfloat16
    )
    
    # 加载 LoRA 权重
    model = PeftModel.from_pretrained(
        base_model,
        "/data/train/output/lora_commodity"
    )
    
    # 合并权重
    merged_model = model.merge_and_unload()
    
    # 保存
    merged_model.save_pretrained("/data/train/output/merged_model")
    tokenizer = AutoTokenizer.from_pretrained("/data/models/Qwen2.5-7B-Instruct")
    tokenizer.save_pretrained("/data/train/output/merged_model")

if __name__ == "__main__":
    merge_model()
```

## 七、模型蒸馏详解

### 7.1 蒸馏原理

```
模型蒸馏 = Teacher-Student 知识迁移

Teacher（大模型）：GPT-4、Claude、Qwen-Max
  ↓ 知识
Student（小模型）：Qwen2.5-7B、Llama-3-8B

蒸馏方法：
1. 响应蒸馏：让小模型学习大模型的输出
2. 特征蒸馏：让小模型学习大模型的中间表示
3. 关系蒸馏：让小模型学习大模型处理问题的模式
```

### 9.2 常见问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| OOM | 显存不足 | 降低 batch_size, 使用 gradient checkpointing |
| loss 不下降 | 学习率不对 | 调整 learning_rate |
| 过拟合 | 数据太少 | 增加数据量, 增加 dropout |
| 训练慢 | batch 太小 | 增加 batch_size, 使用 gradient accumulation |
| 效果差 | 数据质量 | 清洗数据, 增加训练步数 |

### 9.3 训练参数推荐

```
LoRA 推荐配置（7B 模型）：
- rank: 8-16
- alpha: 16-32
- lr: 1e-4 到 3e-4
- batch_size: 4-8
- epochs: 3-5
- warmup: 5-10%

QLoRA 推荐配置（节省显存）：
- batch_size: 1-2
- lr: 2e-4
- 使用 nf4 量化
- 使用 gradient checkpointing
```
