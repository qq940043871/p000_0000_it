## 一、业务价值

```
传统检索 vs 智能检索

传统：
  客户："我要找能用在新能源电池里的钴"
  系统：精确关键词匹配 → 无结果
  结果：客户流失 ❌

智能：
  客户："新能源电池原材料"
  系统：语义理解 → 钴、镍、锂、锰 + 下游应用
  结果：精准匹配 + 关联推荐 ✓
```

| 指标 | 传统检索 | 智能检索 |
|------|---------|---------|
| 召回率 | 40~60% | 85~95% |
| 搜索时长 | 10~30s | <3s |
| 关联推荐点击 | 5% | 25% |
| 转化提升 | — | +30~50% |

### 2.2 典型应用场景

#### ① 智能规格匹配

```markdown
客户需求：需要一批电阻率<0.1的铜杆，用于高端电缆

AI处理流程：
1. 解析需求参数：铜(Cu)、形态=杆、电阻率<0.1
2. 查询知识图谱：
   - 牌号匹配：TU1、TU2 无氧铜
   - 规格匹配：直径8mm/10mm/12mm
   - 替代方案：低氧铜杆(电阻率<0.017)是否可接受
3. 返回结果：
   - 匹配商品列表（含规格对比表）
   - 价格参考
   - 供应商信息
```

#### ② 贸易条款智能解读

```markdown
客户上传：LME铜CIF上海含升水80$/MT

AI解读：

│ 基准价：LME三个月铜（当前$9,215/吨）    │
│ 升水：+$80/吨                          │
│ 到岸价：$9,295/吨 ≈ ¥67,700/吨         │
│ 交货地：上海保税区                      │
│ 备注：含保险，买方自行清关              │
├─────────────────────────────────────────┤
│ ⚠️ 风险提示                            │
│ 当前升水处于历史85%分位，建议观望      │
└─────────────────────────────────────────┘
```

#### ③ 合规自动检查

- **出口管制**：自动识别EAR/瓦森纳清单商品
- **原产地判定**：根据成分构成智能判断原产地
- **环保认证**：RoHS/REACH合规预检

## 四、技术实现

### 4.1 RAG架构设计

```markdown
┌─────────────────────────────────────────────────────────────┐
│                       RAG Pipeline                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  用户Query          检索阶段              生成阶段          │
│  ┌───────┐      ┌───────────┐       ┌────────────┐        │
│  │语义解析│ ──→ │向量化Embedding│ ──→ │LLM生成回答 │        │
│  └───────┘      └───────────┘       └────────────┘        │
│                     │                                       │
│                     ↓                                       │
│              ┌──────────────┐                              │
│              │  混合检索      │                              │
│              │  • 语义向量   │                              │
│              │  • 关键词BM25 │                              │
│              │  • 知识图谱   │                              │
│              └──────────────┘                              │
│                     │                                       │
│                     ↓                                       │
│              ┌──────────────┐                              │
│              │  重排序(Rerank)│                              │
│              │  Cross-Encoder│                              │
│              └──────────────┘                              │
│                     │                                       │
│                     ↓                                       │
│              ┌──────────────┐                              │
│              │  上下文组装    │                              │
│              │  Prompt工程   │                              │
│              └──────────────┘                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 向量化模型选型

| 模型 | 维度 | 优势 | 适用场景 |
|------|-----|------|---------|
| text-embedding-3-large | 3072 | OpenAI官方，效果好 | 通用场景 |
| BGE-M3 | 1024 | 中英双语，中文优化 | 跨境电商 |
| M3E | 768 | 开源免费，部署简单 | 成本敏感 |
| GTE | 1024 | 阿里开源，中文能力强 | 大宗商品 |

### 4.3 知识图谱构建

```python
# 三元组抽取示例
实体识别 = {
    "商品": ["电解铜", "铝锭", "甲醇"],
    "规格": ["99.95%", "φ8mm", "T3"],
    "价格": ["18500元/吨", "CIF上海"],
    "供应商": ["必和必拓", "力拓集团"],
    "仓库": ["上海保税库", "宁波仓库"]
}

关系抽取:
- 电解铜 → [规格为] → 99.95%
- 电解铜 → [仓库位于] → 上海保税库
- 供应商 → [供应] → 电解铜
```
