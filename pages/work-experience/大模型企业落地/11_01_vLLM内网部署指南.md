# vLLM 内网私有化部署指南

> 适用环境：内网隔离、需要 GPU 的高并发推理
> 关联文档：11_00_内网私有化部署开源软件清单.md

---

## 一、部署方式选择

### 1.1 三种部署方式对比

| 方式 | 适用场景 | 优势 | 劣势 |
|------|---------|------|------|
| **Docker 部署** | 快速验证、单机 | 简单快捷 | 性能略低 |
| **裸机部署** | 生产环境、高性能 | 性能最优 | 配置复杂 |
| **Kubernetes** | 集群、弹性扩展 | 可扩展 | 需要 K8s |

### 1.2 推荐配置

```
内网推荐：Docker + 裸金属混合
- 开发测试：Docker
- 生产服务：裸机部署 vLLM
- GPU 资源充足时性能差距可达 10-20%
```

## 二、Docker 部署（推荐）

### 2.1 镜像准备

```bash
# 在有网环境拉取镜像
docker pull vllm/vllm-openai:latest

# 或指定 CUDA 版本
docker pull vllm/vllm-openai:latest-cuda12.1

# 导出镜像
docker save vllm/vllm-openai:latest -o vllm-openai.tar

# 传输到内网服务器
scp vllm-openai.tar user@内网服务器:/data/images/

# 内网加载镜像
docker load -i /data/images/vllm-openai.tar
```

### 2.2 模型文件准备

```bash
# 方式1：直接使用 Huggingface 格式
# 下载到本地后上传
scp -r Qwen2.5-7B-Instruct/ user@内网:/data/models/llm/

# 方式2：使用safetensors格式（推荐，更快更安全）
# 需要模型以 safetensors 格式存储

# 目录结构
/data/models/llm/
├── Qwen2.5-7B-Instruct/
│   ├── config.json
│   ├── model.safetensors (或 pytorch_model.bin)
│   ├── tokenizer.json
│   └── ...
```

### 2.3 Docker Compose 配置

```yaml
# docker-compose-vllm.yml
version: '3.8'

services:
  vllm:
    image: vllm/vllm-openai:latest
    container_name: vllm-server
    restart: unless-stopped
    ports:
      - "8000:8000"      # API 端口
      - "8001:8001"      # Metrics 端口
    volumes:
      - /data/models:/models          # 模型目录
      - /data/huggingface:/root/.huggingface  # 缓存
    environment:
      - CUDA_VISIBLE_DEVICES=0,1       # GPU 设备号
      - VLLM_WORKER_MULTIPROC_METHOD=spawn
      - HF_HOME=/root/.huggingface
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 2                  # GPU 数量
              capabilities: [gpu]
    command: >
      --model /models/llm/Qwen2.5-7B-Instruct
      --tensor-parallel-size 2
      --gpu-memory-utilization 0.9
      --max-model-len 8192
      --port 8000
      --host 0.0.0.0
      --trust-remote-code
      --enforce-eager                   # 内网禁用 CUDA graph
```

### 2.4 启动服务

```bash
# 启动
docker-compose -f docker-compose-vllm.yml up -d

# 查看日志
docker-compose -f docker-compose-vllm.yml logs -f

# 检查状态
docker ps | grep vllm
```

## 三、裸机部署（生产环境）

### 3.1 系统环境准备

```bash
# 1. 安装 NVIDIA Driver
apt-get update
apt-get install -y nvidia-driver-535

# 2. 安装 CUDA Toolkit
wget https://developer.download.nvidia.com/compute/cuda/12.1.0/local_installers/cuda_12.1.0_530.30.02_linux.run
sudo sh cuda_12.1.0_530.30.02_linux.run

# 3. 安装 NVIDIA Container Toolkit
distribution=$(. /etc/os-release;echo $ID$VERSION_ID)
curl -fsSL https://nvidia.github.io/nvidia-docker/gpgkey | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg
curl -s -L https://nvidia.github.io/nvidia-container-toolkit/$distribution/nvidia-container-toolkit.list | \
    sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | \
    sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list
apt-get update
apt-get install -y nvidia-container-toolkit
nvidia-ctk runtime configure --runtime=docker
systemctl restart docker
```

### 3.2 vLLM 安装

```bash
# 方式1：pip 安装
pip install vllm

# 方式2：从源码编译（获取最新优化）
git clone https://github.com/vllm-project/vllm.git
cd vllm
pip install -e .
```

### 3.3 启动脚本

```bash
#!/bin/bash
# start-vllm.sh

export CUDA_VISIBLE_DEVICES=0,1
export VLLM_WORKER_MULTIPROC_METHOD=spawn

python -m vllm.entrypoints.openai.api_server \
    --model /data/models/llm/Qwen2.5-7B-Instruct \
    --tensor-parallel-size 2 \
    --gpu-memory-utilization 0.9 \
    --max-model-len 8192 \
    --port 8000 \
    --host 0.0.0.0 \
    --trust-remote-code \
    --served-model-name Qwen2.5-7B \
    --max-num-batched-seqs 32 \
    --max-num-seqs 256 \
    >> /data/logs/vllm.log 2>&1 &

echo "vLLM 服务已启动，PID: $!"
```

## 五、生产配置

### 5.1 systemd 服务配置

```bash
# /etc/systemd/system/vllm.service
[Unit]
Description=vLLM OpenAI API Server
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/data
ExecStart=/usr/bin/python3 -m vllm.entrypoints.openai.api_server \
    --model /data/models/llm/Qwen2.5-7B-Instruct \
    --tensor-parallel-size 2 \
    --gpu-memory-utilization 0.9 \
    --max-model-len 8192 \
    --port 8000 \
    --host 0.0.0.0 \
    --trust-remote-code
Restart=always
RestartSec=10
StandardOutput=append:/data/logs/vllm.log
StandardError=append:/data/logs/vllm.log

[Install]
WantedBy=multi-user.target
```

```bash
# 启用服务
systemctl daemon-reload
systemctl enable vllm
systemctl start vllm
systemctl status vllm
```

### 5.2 Nginx 反向代理

```nginx
# /etc/nginx/conf.d/vllm.conf
upstream vllm_backend {
    server 127.0.0.1:8000;
}

server {
    listen 8000;
    server_name _;
    
    client_max_body_size 10M;
    
    location / {
        proxy_pass http://vllm_backend;
        proxy_http_version 1.1;
        
        # 超时配置
        proxy_connect_timeout 60s;
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
        
        # 流式响应
        proxy_buffering off;
        proxy_cache off;
        chunked_transfer_encoding on;
    }
}
```

## 六、常见问题

### 6.1 显存不足 OOM

```bash
# 解决1：降低显存使用
--gpu-memory-utilization 0.7

# 解决2：减小最大长度
--max-model-len 4096

# 解决3：使用量化
--quantization fp8
```

### 6.2 模型加载慢

```bash
# 内网建议禁用在线下载
-- hf_transfer False
# 确保模型文件完整在本地
```

### 6.3 性能下降

```bash
# 检查 GPU 利用率
nvidia-smi

# 检查推理日志
tail -f /data/logs/vllm.log

# 常见原因：
# 1. 并发过高 - 降低 -max-num-seqs
# 2. 显存碎片 - 重启服务
# 3. CPU 瓶颈 - 增加 CPU 核心
```
