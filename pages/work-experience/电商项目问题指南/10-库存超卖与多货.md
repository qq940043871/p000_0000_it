# 10 - 库存超卖与多货

> 电商核心问题：如何保证库存不超卖、支持多货

---

## 📖 概述

库存超卖是电商系统的核心技术难点。"多货"是指用户下单后，系统能准确预留并扣减库存，保证订单可履约。本章详细介绍库存管理的完整方案。

---

## ⚠️ 一、问题分析

### 1.1 什么是库存超卖

```
库存超卖场景：

商品库存: 1件
用户A: ──► 下单 ──► 扣减库存(1-1=0) ──► 成功 ✅
用户B: ──► 下单 ──► 扣减库存(0-1=-1) ──► 超卖! ❌
```

### 1.2 超卖后果

| 后果 | 影响 |
|------|------|
| 订单无法履约 | 用户投诉、赔偿 |
| 退款率高 | 平台信誉损失 |
| 库存负数 | 财务对账异常 |
| 资损 | 赔付近百万元 |

### 1.3 电商库存类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **可用库存** | 真实可售库存 | 100件 |
| **冻结库存** | 预占未支付订单 | 下单锁定10分钟 |
| **已售库存** | 已支付订单 | 真正售出 |
| **在途库存** | 采购/调拨中 | 仓库间调拨 |

```
库存模型：
可用库存 = 总库存 - 冻结库存 - 已售库存 + 在途库存
```

## 🏗️ 二、库存扣减方案

### 2.1 方案一：数据库乐观锁

```java
// 乐观锁扣减库存
@Transactional
public boolean deductStock(Long skuId, Integer quantity) {
    // 1. 先查询当前库存
    Sku sku = skuDao.selectById(skuId);
    
    // 2. 检查库存是否充足
    if (sku.getStock() < quantity) {
        throw new StockException("库存不足");
    }
    
    // 3. 乐观锁更新：WHERE stock >= quantity
    int rows = skuDao.updateStockWithVersion(
        skuId, 
        sku.getVersion(),
        sku.getStock() - quantity
    );
    
    return rows > 0;
}

@Mapper
public interface SkuDao {
    
    @Update("UPDATE t_sku SET stock = stock - #{quantity}, " +
            "version = version + 1 " +
            "WHERE id = #{id} AND stock >= #{quantity} AND version = #{version}")
    int updateStockWithVersion(@Param("id") Long id, 
                              @Param("version") Long version,
                              @Param("quantity") Integer quantity);
}
```

**优点**：简单、可靠
**缺点**：高并发时失败率高

### 2.2 方案二：Redis预扣 + 数据库最终一致性

**这是大厂主流方案！**

```
下单流程：

1. 用户点击下单
       │
       ▼
2. Redis预扣库存 (DECR)
       │
       ├──► 成功 ──► 创建订单 ──► 异步写DB ──► 返回成功
       │
       └──► 失败 ──► 库存不足 ──► 返回失败
```

**Redis预扣代码**：

```lua
-- Redis Lua脚本：库存扣减
-- KEYS[1] = 库存key，如 "stock:sku:10001"
-- ARGV[1] = 扣减数量

local stock = redis.call('GET', KEYS[1])

if stock == false then
    return -1  -- 库存key不存在
end

local stockNum = tonumber(stock)
local deductNum = tonumber(ARGV[1])

if stockNum < deductNum then
    return 0  -- 库存不足
end

-- 扣减库存
redis.call('DECRBY', KEYS[1], deductNum)

return stockNum - deductNum  -- 返回扣减后的库存
```

**Java调用**：

```java
@Service
public class StockService {
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    @Autowired
    private SkuDao skuDao;
    
    private static final String STOCK_KEY_PREFIX = "stock:sku:";
    
    /**
     * 预扣库存（Redis）
     */
    public boolean preDeductStock(Long skuId, Integer quantity) {
        String key = STOCK_KEY_PREFIX + skuId;
        
        // 执行Lua脚本，保证原子性
        Long result = redisTemplate.execute(
            new DefaultRedisScript<>(STOCK_LUA, Long.class),
            Collections.singletonList(key),
            String.valueOf(quantity)
        );
        
        return result != null && result >= 0;
    }
    
    /**
     * 库存扣减失败，回补Redis
     */
    public void revertStock(Long skuId, Integer quantity) {
        String key = STOCK_KEY_PREFIX + skuId;
        redisTemplate.opsForValue().increment(key, quantity);
    }
    
    /**
     * 异步同步到数据库
     */
    @Async
    @Transactional
    public void syncToDatabase(Long skuId, Integer quantity) {
        // 数据库乐观锁扣减
        skuDao.decrementStock(skuId, quantity);
        
        // 记录库存流水
        stockFlowDao.insert(StockFlow.builder()
            .skuId(skuId)
            .type(1)  // 1=扣减
            .quantity(quantity)
            .build());
    }
}
```

### 2.3 方案三：分布式锁（最严格）

```java
@Service
public class StockLockService {
    
    @Autowired
    private RedissonClient redisson;
    
    @Autowired
    private SkuDao skuDao;
    
    /**
     * 分布式锁扣减库存
     */
    public boolean deductStockWithLock(Long skuId, Integer quantity) {
        String lockKey = "lock:stock:" + skuId;
        RLock lock = redisson.getLock(lockKey);
        
        try {
            // 获取锁，最多等待5秒，锁定10秒后自动释放
            if (lock.tryLock(5, 10, TimeUnit.SECONDS)) {
                try {
                    // 查询库存
                    Sku sku = skuDao.selectById(skuId);
                    if (sku.getStock() < quantity) {
                        return false;  // 库存不足
                    }
                    
                    // 扣减库存
                    sku.setStock(sku.getStock() - quantity);
                    skuDao.updateById(sku);
                    
                    return true;
                } finally {
                    lock.unlock();
                }
            }
            return false;  // 获取锁失败
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            return false;
        }
    }
}
```

## 🔄 三、定时巡检与补偿

### 3.1 库存对账机制

```java
/**
 * 库存定时对账
 * 每5分钟执行一次，检查Redis和DB的库存一致性
 */
@Scheduled(cron = "0 */5 * * * ?")
public void stockReconciliation() {
    List<Sku> allSku = skuDao.selectAll();
    
    for (Sku sku : allSku) {
        // 1. 查询Redis库存
        String redisKey = "stock:sku:" + sku.getId();
        String redisStock = redisTemplate.opsForValue().get(redisKey);
        int redisQty = redisStock == null ? 0 : Integer.parseInt(redisStock);
        
        // 2. 查询DB可用库存
        int dbStock = sku.getStock();
        
        // 3. 差异大于阈值告警
        if (Math.abs(redisQty - dbStock) > TOLERANCE) {
            alertService.send("库存不一致: skuId=" + sku.getId() 
                + ", redis=" + redisQty + ", db=" + dbStock);
        }
    }
}
```

@Scheduled(cron = "0 0 * * * ?")

        }
    }
}

## 📊 四、多货实现原理

### 4.1 什么是"多货"

```
"多货" ≠ 超卖
"多货" = 库存预占 + 订单保障

多货场景：
1. 商品有100件库存
2. 10个用户同时下单，各买1件
3. 10个订单全部创建成功
4. 10个订单都能正常履约（发货）

实现方式：
1. 库存预扣：下单时预占库存
2. 库存锁定期：未支付订单库存冻结
3. 库存释放：订单取消/超时释放
```

### 4.2 库存生命周期

```
库存状态流转：

[可用] ──下单──► [冻结] ──支付──► [锁定] ──发货──► [已售]
   ▲                    │                            │
   │                    │ 取消/超时                   │ 退款
   │                    ▼                            ▼
   └──────────── [释放] ◄───────────────────── [退回]
```

### 4.3 完整扣减流程

```java
@Transactional
public Order createOrder(OrderDTO dto) {
    // 1. 校验商品状态
    Sku sku = skuService.getById(dto.getSkuId());
    if (sku == null || sku.getStatus() != 1) {
        throw new BizException("商品已下架");
    }
    
    // 2. 校验活动库存（如果参加活动）
    if (dto.getActivityId() != null) {
        boolean available = activityService.checkStock(
            dto.getActivityId(), 
            dto.getQuantity()
        );
        if (!available) {
            throw new BizException("活动库存不足");
        }
    }
    
    // 3. Redis预扣库存（防止超卖）
    boolean stockOk = stockService.preDeductStock(
        dto.getSkuId(), 
        dto.getQuantity()
    );
    if (!stockOk) {
        throw new BizException("库存不足");
    }
    
    try {
        // 4. 创建订单
        Order order = Order.builder()
            .userId(dto.getUserId())
            .skuId(dto.getSkuId())
            .quantity(dto.getQuantity())
            .status(OrderStatus.CREATED)
            .build();
        orderDao.insert(order);
        
        // 5. 发送延迟消息（超时未支付自动释放）
        mqProducer.sendDelayMessage(
            "order-timeout",
            new OrderTimeoutMessage(order.getId()),
            15 * 60 * 1000  // 15分钟后检查
        );
        
        return order;
        
    } catch (Exception e) {
        // 扣减失败，回补Redis
        stockService.revertStock(dto.getSkuId(), dto.getQuantity());
        throw e;
    }
}

/**
 * 支付回调处理
 */
public void onOrderPaid(Long orderId) {
    Order order = orderDao.selectById(orderId);
    
    // 1. 更新订单状态
    order.setStatus(OrderStatus.PAID);
    orderDao.updateById(order);
    
    // 2. 发送订单已支付消息
    mqProducer.send("order-paid", order);
    
    // 3. 锁定库存（从冻结转为正式扣减）
    stockService.confirmDeduct(order.getSkuId(), order.getQuantity());
}

/**
 * 取消/超时释放库存
 */
public void releaseStock(Long orderId) {
    Order order = orderDao.selectById(orderId);
    
    if (order.getStatus() == OrderStatus.PAID) {
        return;  // 已支付的不能释放
    }
    
    // 1. 更新订单状态
    order.setStatus(OrderStatus.CANCELLED);
    orderDao.updateById(order);
    
    // 2. 释放冻结的库存
    stockService.releaseFrozenStock(order.getSkuId(), order.getQuantity());
    
    // 3. 恢复活动库存
    if (order.getActivityId() != null) {
        activityService.releaseStock(
            order.getActivityId(),
            order.getQuantity()
        );
    }
}
```

## ⚠️ 五、常见问题与解决方案

### 5.1 高并发扣库存失败率高

**问题**：纯数据库乐观锁，高并发下大量更新失败

**解决**：
1. Redis预扣作为第一道防线
2. 失败后进入排队队列
3. 重试机制

### 5.2 Redis和DB数据不一致

**问题**：Redis扣减成功，DB更新失败

**解决**：
1. 记录库存流水日志
2. 定时对账修复
3. 异常订单人工处理

### 5.3 超时订单库存释放

**问题**：用户超时未支付，库存未及时释放

**解决**：
1. MQ延迟消息触发释放
2. 每分钟扫描超时订单
3. 释放前检查订单状态

### 5.4 下单成功后支付失败

**问题**：库存已预扣，但支付失败

**解决**：
1. 支付失败不立即释放
2. 用户可重试支付
3. 15分钟超时自动释放

## 🛠️ 七、核心代码模板

### 7.1 Redis Lua脚本（完整版）

```lua
-- 库存扣减脚本 v2.0
-- KEYS[1]: 库存key，如 stock:sku:10001
-- KEYS[2]: 订单锁定key，如 order:lock:sku:10001
-- ARGV[1]: 扣减数量
-- ARGV[2]: 锁定过期时间（秒）

local stock = redis.call('GET', KEYS[1])
if stock == false then
    return -1  -- 商品不存在
end

local stockNum = tonumber(stock)
local deductNum = tonumber(ARGV[1])

if stockNum < deductNum then
    return 0  -- 库存不足
end

-- 扣减库存
redis.call('DECRBY', KEYS[1], deductNum)

-- 记录订单锁定
local lockKey = KEYS[2] .. ':' .. ARGV[3]  -- 加上订单ID
redis.call('SETEX', lockKey, ARGV[2], deductNum)

return stockNum - deductNum
```
