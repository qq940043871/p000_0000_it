# 14 - 接口性能优化

> 如何让接口响应时间从秒级降到毫秒级

---

## 📖 概述

接口性能直接影响用户体验。本章介绍SQL优化、缓存、异步等核心技术。

---

## 🗄️ 一、SQL优化

### 1.1 索引优化

```sql
-- 常见慢查询优化

-- 1. 覆盖索引：避免回表
SELECT id, name FROM t_user WHERE phone = '13800138000';
-- 覆盖索引: (phone) INCLUDE (name)

-- 2. 联合索引：最左前缀原则
CREATE INDEX idx_user_status_created ON t_order(user_id, status, created_at);
-- 可优化: WHERE user_id = ? AND status = ? AND created_at > ?

-- 3. 避免索引失效
-- ❌ WHERE status = 1 AND name LIKE '%关键字%'
-- ✅ WHERE status = 1 AND name LIKE '关键字%'
```

### 1.2 分页优化

```sql
-- ❌ 深分页问题
SELECT * FROM t_order LIMIT 1000000, 10;

-- ✅ 游标分页
SELECT * FROM t_order WHERE id > #{lastId} ORDER BY id LIMIT 10;

-- ✅ 延迟关联
SELECT o.* FROM t_order o 
INNER JOIN (SELECT id FROM t_order ORDER BY id LIMIT 1000000, 10) t ON o.id = t.id;
```

## ⚡ 二、异步处理

### 2.1 异步化示例

```java
// @Async异步执行
@Async("asyncExecutor")
public CompletableFuture<OrderDetail> getOrderDetail(Long orderId) {
    Order order = orderService.getById(orderId);
    User user = userService.getById(order.getUserId());
    Product product = productService.getById(order.getProductId());
    
    return CompletableFuture.completedFuture(
        new OrderDetail(order, user, product)
    );
}
```
