## 📈 二、Prometheus配置

### 2.1 指标暴露

```java
@RestController
public class MetricsController {
    
    @Autowired
    private Counter orderCounter;
    
    @GetMapping("/api/orders")
    public List<Order> getOrders() {
        orderCounter.increment();
        return orderService.list();
    }
}

// Spring Boot Actuator + Micrometer
management:
  endpoints:
    web:
```

### 3.2 日志规范

```java
// 日志规范
@Slf4j
public class OrderService {
    
    public void createOrder(OrderDTO dto) {
        // 使用MDC追踪
        MDC.put("traceId", TraceContext.getTraceId());
        MDC.put("orderId", dto.getOrderId());
        
        try {
            log.info("创建订单, userId={}, skuId={}", dto.getUserId(), dto.getSkuId());
            // 业务逻辑
        } catch (Exception e) {
            log.error("创建订单失败, userId={}", dto.getUserId(), e);
            throw e;
        } finally {
            MDC.clear();
        }
    }
}
```

## 🚨 四、告警规则

```yaml
# AlertManager告警规则
groups:
  - name: ecommerce_alerts
    rules:
      - alert: HighErrorRate
        expr: rate(http_requests_total{status=~"5.."}[5m]) > 0.01
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "接口错误率超过1%"
      
      - alert: HighLatency
        expr: histogram_quantile(0.95, http_request_duration_seconds[5m]) > 0.5
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "P95响应时间超过500ms"
```
