# 07 - 数据库选型与分库分表

> 如何选择数据库并实现水平扩展

---

## 📖 概述

数据库是电商系统的核心数据存储。本章介绍数据库选型策略和分库分表实践。

---

## 🗄️ 一、数据库选型

### 1.1 电商数据库需求

| 需求 | 说明 |
|------|------|
| **事务强一致** | 订单、支付必须ACID |
| **高并发读写** | 商品查询、订单创建 |
| **海量数据存储** | 历史订单、用户行为 |
| **复杂查询** | 多维度筛选、报表 |

### 1.2 选型对比

| 数据库 | 优势 | 劣势 | 适用场景 |
|--------|------|------|----------|
| **MySQL** | 成熟、事务、运维 | 并发有限 | 核心业务 |
| **PostgreSQL** | 功能强、扩展性 | 生态稍弱 | 复杂查询 |
| **MongoDB** | 文档型、扩展 | 事务弱 | 商品属性 |
| **TiDB** | NewSQL、水平扩展 | 成本高 | HTAP场景 |

## 📊 二、分库分表

### 2.1 分片策略

```yaml
# ShardingSphere 分片配置
spring:
  shardingsphere:
    rules:
      sharding:
        tables:
          t_order:
            # 按用户ID分片
            actualDataNodes: ds_${0..3}.t_order_${user_id % 4}
            databaseStrategy:
              standard:
                shardingColumn: user_id
                shardingAlgorithmName: database_inline
            tableStrategy:
              standard:
                shardingColumn: order_id
                shardingAlgorithmName: order_inline
```

### 2.2 分片算法

| 算法 | 公式 | 特点 |
|------|------|------|
| **Hash** | `id % n` | 均匀分布 |
| **Range** | `id BETWEEN` | 范围查询优 |
| **时间** | `DATE(created)` | 历史数据分离 |

## 🔄 三、读写分离

### 3.1 架构

```
┌────────────┐
│  应用服务  │
└─────┬──────┘
      │
      ├──► 写 ──► Master (主库)
      │
      └──► 读 ──► Slave1 / Slave2 (从库)
```

### 3.2 配置示例

```java
@Configuration
public class DataSourceConfig {
    
    @Bean
    public DataSource dataSource() {
        HikariDataSource master = createDataSource("jdbc:mysql://master:3306", "master");
        HikariDataSource slave1 = createDataSource("jdbc:mysql://slave1:3306", "slave1");
        HikariDataSource slave2 = createDataSource("jdbc:mysql://slave2:3306", "slave2");
        
        // 路由策略: 读走从库，写走主库
        ReplicationModeConfiguration config = 
            new ReplicationModeConfiguration("master", Arrays.asList(slave1, slave2));
        
        return new ReplicationDataSource(config);
    }
}
```
