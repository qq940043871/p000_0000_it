## ⚠️ 三、缓存问题

### 3.1 缓存穿透

**问题**: 请求的数据既不在缓存也不在数据库，导致大量请求打到数据库。

**解决方案**:
```java
public Product getProduct(Long id) {
    String cacheKey = "product:" + id;
    
    // 1. 查询缓存
    Product product = redisTemplate.opsForValue().get(cacheKey);
    
    // 2. 缓存命中
    if (product != null) {
        return product;
    }
    
    // 3. 查询数据库
    product = productDao.selectById(id);
    
    // 4. 数据库也没有，返回空并写入空值标记
    if (product == null) {
        // 写入空值缓存，防止穿透
        redisTemplate.opsForValue().set(cacheKey, "", 5, TimeUnit.MINUTES);
        return null;
    }
    
    // 5. 写入缓存
    redisTemplate.opsForValue().set(cacheKey, product, 30, TimeUnit.MINUTES);
    return product;
}
```

### 3.2 缓存雪崩

**问题**: 大量缓存同时过期，导致瞬时数据库压力过大。

**解决方案**:
```java
@Configuration
public class RedisConfig {
    
    @Bean
    public RedisTemplate<String, Object> redisTemplate() {
        RedisTemplate<String, Object> template = new RedisTemplate<>();
        
        // 设置过期时间增加随机值，防止雪崩
        template.setDefaultSerializer(new Jackson2JsonRedisSerializer<>(Object.class));
        
        return template;
    }
}

// 使用: 过期时间 = 基准时间 + 随机时间
public void setWithRandomExpire(String key, Object value, long baseSeconds) {
    int randomSeconds = ThreadLocalRandom.current().nextInt(60);
    redisTemplate.opsForValue().set(key, value, baseSeconds + randomSeconds, TimeUnit.SECONDS);
}
```

### 3.3 缓存击穿

**问题**: 热点key过期瞬间，大量请求涌入数据库。

**解决方案 - 互斥锁**:
```java
public Product getProductWithLock(Long id) {
    String cacheKey = "product:" + id;
    
    // 1. 先查缓存
    Product product = redisTemplate.opsForValue().get(cacheKey);
    if (product != null) {
        return product;
    }
    
    // 2. 获取分布式锁
    String lockKey = "lock:product:" + id;
    String lockValue = UUID.randomUUID().toString();
    
    boolean locked = redisTemplate.opsForValue()
        .setIfAbsent(lockKey, lockValue, 10, TimeUnit.SECONDS);
    
    if (locked) {
        try {
            // 3. 双重检查
            product = redisTemplate.opsForValue().get(cacheKey);
            if (product != null) {
                return product;
            }
            
            // 4. 查询数据库
            product = productDao.selectById(id);
            
            // 5. 写入缓存
            if (product != null) {
                redisTemplate.opsForValue().set(cacheKey, product, 30, TimeUnit.MINUTES);
            }
            
            return product;
        } finally {
            // 6. 释放锁
            redisTemplate.delete(lockKey);
        }
    } else {
        // 7. 等待后重试
        try {
            Thread.sleep(100);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
        return getProductWithLock(id);
    }
}
```

### 3.4 缓存一致性

**Cache Aside 模式**:
```
读操作:
1. 读缓存 -> 命中 -> 返回
2. 读缓存 -> 未命中 -> 读数据库 -> 写缓存 -> 返回

写操作:
1. 写数据库 -> 删除缓存 (不是更新缓存!)
```

## 💾 四、热点数据处理

### 4.1 热点Key探测

```python
# 基于Redis的热点Key探测
def detect_hot_keys(redis_client):
    # 采样统计
    hot_keys = []
    
    # 使用MONITOR采样
    for i in range(100000):
        key = redis_client.randomkey()
        if key.startswith("product:") or key.startswith("sku:"):
            hot_keys.append(key)
    
    # 统计频率
    from collections import Counter
    counter = Counter(hot_keys)
    
    # 返回Top100热点
    return counter.most_common(100)
```

### 4.2 热点Key打散

```java
// 热点Key加随机后缀
public String getHotKey(String key) {
    // 探测为热点Key后，自动加后缀打散
    int suffix = ThreadLocalRandom.current().nextInt(10);
    return key + ":" + suffix;
}
```
