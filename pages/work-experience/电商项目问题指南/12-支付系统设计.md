# 12 - 支付系统设计

> 如何设计安全可靠的支付系统

---

## 📖 概述

支付系统是电商资金安全的核心。本章介绍支付流程、风控、对账等核心设计。

---

## 💰 一、支付流程

### 1.1 聚合支付架构

```
用户下单 ──► 选择支付方式 ──► 发起支付 ──► 跳转支付渠道
                                                        │
                                                    支付成功/失败
                                                        │
┌──────────────────────────────────────────────────────┘
│
├──► 异步回调通知 ◄── 支付宝/微信
│
├──► 更新订单状态
│
├──► 发送支付成功消息
│
└──► 触发后续流程（发货等）
```

## 🔒 二、幂等处理

### 2.1 支付幂等设计

```java
@Service
public class PaymentService {
    
    /**
     * 支付回调处理（幂等）
     */
    public void handlePayCallback(PayCallbackDTO callback) {
        String idempotentKey = "pay:callback:" + callback.getTradeNo();
        
        // 1. 幂等检查
        Boolean success = redisTemplate.opsForValue()
            .setIfAbsent(idempotentKey, "processing", 24, TimeUnit.HOURS);
        
        if (!success) {
            log.info("支付回调已处理: {}", callback.getTradeNo());
            return;
        }
        
        try {
            // 2. 校验签名
            if (!verifySign(callback)) {
                throw new BizException("签名校验失败");
            }
            
            // 3. 查询订单
            Order order = orderService.getByOrderNo(callback.getOrderNo());
            if (order == null) {
                log.warn("订单不存在: {}", callback.getOrderNo());
                return;
            }
            
            // 4. 幂等：检查订单状态
            if (order.getPayStatus() == PayStatus.PAID) {
                log.info("订单已支付: {}", callback.getOrderNo());
                return;
            }
            
            // 5. 更新订单支付状态
            orderService.updatePayStatus(order.getId(), PayStatus.PAID, callback.getTradeNo());
            
            // 6. 发送支付成功消息
            mqProducer.send("order-paid", order);
            
        } catch (Exception e) {
            log.error("支付回调处理异常", e);
            throw e;
        }
    }
}
```

## 📊 三、对账系统

### 3.1 对账流程

```
T+1日 09:00
      │
      ▼
┌─────────────────┐
│  下载对账单      │ ◄── 支付宝/微信提供
│  (商户平账)    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  生成内部对账单  │ ◄── 按订单维度汇总
│  (T日订单)     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  比对差异       │
│                 │
│  内部有，渠道无 │ ◄── 疑似掉单
│  渠道有，内部无 │ ◄── 疑似多付
│  金额不一致     │ ◄── 疑似金额错误
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  生成差异报表    │
│  人工介入处理   │
└─────────────────┘
```
