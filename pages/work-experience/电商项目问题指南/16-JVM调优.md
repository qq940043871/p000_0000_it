# 16 - JVM调优

> 如何让Java应用更稳定高效

## 📖 概述

JVM调优是高并发Java系统的必备技能。本章介绍内存模型、垃圾回收、参数调优。

## 🗑️ 二、垃圾回收

### 2.1 垃圾回收器对比

| 回收器 | 适用区域 | 算法 | 特点 |
|--------|----------|------|------|
| **Serial** | 年轻代 | 复制 | 单线程，简单 |
| **ParNew** | 年轻代 | 复制 | 多线程，停顿长 |
| **Parallel Scavenge** | 年轻代 | 复制 | 吞吐量优先 |
| **CMS** | 老年代 | 标记-清除 | 并发，低停顿 |
| **G1** | 全部 | 标记-整理 | 可预测停顿 |
| **ZGC** | 全部 | 着色指针 | <1ms停顿 |

### 2.2 推荐配置

```bash
# 高并发电商应用JVM参数
java -Xms4g -Xmx4g \
     -Xmn2g \
     -XX:+UseG1GC \
     -XX:MaxGCPauseMillis=200 \
     -XX:+HeapDumpOnOutOfMemoryError \
     -XX:HeapDumpPath=/var/logs/oom.hprof \
     -Xloggc:/var/logs/gc.log \
     -XX:+PrintGCDetails \
     -XX:+PrintGCDateStamps
```

## ⚠️ 三、常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| OOM | 内存泄漏 | dump分析 |
| FGC频繁 | 内存分配不合理 | 调整比例 |
| 线程阻塞 | GC停顿 | G1/ZGC |
