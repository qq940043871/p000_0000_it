# 06 - CDN与静态资源

> 如何让用户就近访问、实现动静分离

---

## 📖 概述

CDN（内容分发网络）是前端性能优化的关键。本章介绍CDN架构和静态资源管理。

---

## 🏗️ 一、CDN架构

### 1.1 CDN工作原理

```
用户请求 ──► 浏览器 ──► DNS解析 ──► CDN DNS ──► 就近调度
                                                    │
                                              边缘节点缓存
                                                    │
                                              命中: 直接返回
                                                    │
                                         未命中: 回源获取
                                                    │
                                              源站服务器
```

### 1.2 CDN节点分布

| 层级 | 节点类型 | 数量 | 覆盖范围 |
|------|----------|------|----------|
| L1边缘 | 省级节点 | 100+ | 省会城市 |
| L2中心 | 区域节点 | 30+ | 华东/华南等 |
| L3骨干 | 核心节点 | 10+ | 北上广深 |

---

## 🔧 二、静态资源优化

### 2.1 资源分类

| 类型 | 缓存策略 | 示例 |
|------|----------|------|
| **静态资源** | 永久缓存+版本化 | JS/CSS/图片 |
| **动态资源** | 不缓存或短缓存 | HTML/JSON |
| **用户资源** | 私有缓存 | 用户头像 |

### 2.2 缓存策略

```nginx
# Nginx配置示例
location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
    expires 1y;                          # 缓存1年
    add_header Cache-Control "public, immutable";
    access_log off;                       # 关闭日志
}

location ~* \.(html|json|xml)$ {
    expires -1;
    add_header Cache-Control "no-cache, no-store";
}
```

## ⚡ 三、动静分离

### 3.1 分离架构

```
┌─────────────────────────────────────────────────────────┐
│                      用户请求                               │
└─────────────────────────────────────────────────────────┘
                         │
          ┌──────────────┼──────────────┐
          │              │              │
          ▼              ▼              ▼
    ┌───────────┐  ┌───────────┐  ┌───────────┐
    │  静态请求  │  │  动态请求  │  │  接口请求  │
    │  (CDN)    │  │  (Nginx)   │  │  (Gateway) │
    └─────┬─────┘  └─────┬─────┘  └─────┬─────┘
          │               │               │
          ▼               ▼               ▼
    ┌───────────┐  ┌───────────┐  ┌───────────┐
    │ OSS/FastDFS│  │   Tomcat  │  │  微服务   │
    │  存储      │  │  应用      │  │  集群     │
    └───────────┘  └───────────┘  └───────────┘
```

### 3.2 文件服务选型

| 方案 | 适用规模 | 特点 |
|------|----------|------|
| **NFS** | < 100TB | 简单、低成本 |
| **FastDFS** | 100TB-1PB | 高并发、分组 |
| **OSS/MinIO** | PB级+ | 云原生、托管 |
