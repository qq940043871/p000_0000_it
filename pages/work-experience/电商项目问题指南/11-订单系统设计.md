# 11 - 订单系统设计

## 📖 概述

订单系统是电商交易的核心。本章介绍订单状态机、幂等性、分布式ID等核心设计。

## 🏗️ 一、订单状态机

### 1.1 订单状态流转

```
[待支付] ──支付成功──► [已支付] ──发货──► [已发货] ──确认收货──► [已完成]
    │                          │                          │
    │ 超时/取消                │ 退款申请                  │ 申请售后
    ▼                          ▼                          ▼
[已取消]                   [退款中]                    [售后中]
                              │
                              ▼
                         [已退款]
```

### 1.2 状态机实现

```java
public enum OrderStatus {
    CREATED(100, "待支付"),
    PAID(200, "已支付"),
    SHIPPED(300, "已发货"),
    CONFIRMED(400, "已完成"),
    CANCELLED(500, "已取消"),
    REFUNDING(600, "退款中"),
    REFUNDED(700, "已退款");
    
    // 状态转换校验
    public boolean canTransitionTo(OrderStatus target) {
        return TRANSITIONS.get(this).contains(target);
    }
}
```

## 🔑 二、幂等性设计

### 2.1 Token机制

```java
@Service
public class IdempotentService {
    
    public String generateToken() {
        return UUID.randomUUID().toString();
    }
    
    public boolean checkToken(String token) {
        String key = "idempotent:token:" + token;
        Boolean result = redisTemplate.opsForValue().setIfAbsent(key, "1", 10, TimeUnit.MINUTES);
        return Boolean.TRUE.equals(result);
    }
}

// 接口使用
@PostMapping("/order/create")
public Result<Order> createOrder(@RequestBody OrderDTO dto,
                                 @RequestHeader("X-Idempotent-Token") String token) {
    // 1. 校验幂等Token
    if (!idempotentService.checkToken(token)) {
        return Result.fail("请勿重复提交");
    }
    
    // 2. 业务处理
    return orderService.create(dto);
}
```

## 🆔 三、分布式ID

### 3.1 Snowflake算法

```java
public class SnowflakeIdWorker {
    private final long twepoch = 1609459200000L;  // 2021-01-01
    private final long workerIdBits = 5L;
    private final long datacenterIdBits = 5L;
    
    private long workerId;
    private long datacenterId;
    private long sequence = 0L;
    
    public long nextId() {
        long timestamp = System.currentTimeMillis() - twepoch;
        long sequenceMask = ~(-1L << 12L);
        
        return (timestamp << 22) | (datacenterId << 17) | (workerId << 12) | sequence++;
    }
}
```
