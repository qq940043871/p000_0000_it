## 🔥 核心问题速查

### Q: 多货是怎么实现的？
**A**: 详见 [库存超卖与多货](./sections/10-库存超卖与多货.md)

核心思路：
1. **库存预占**：下单时预占库存，支付后真正扣减
2. **锁机制**：Redis分布式锁 + 数据库乐观锁双重保障
3. **消息队列**：异步解耦，防止并发冲击
4. **对账机制**：日终对账，异常订单自动补偿

### Q: 怎么保证数据不丢失？
**A**: 详见 [数据备份与恢复](./sections/09-数据备份与恢复.md)

核心策略：
1. **多副本存储**：主从复制 + 跨机房同步
2. **写前日志(WAL)**：事务日志实时落盘
3. **定期快照**：全量备份 + 增量备份
4. **异地容灾**：跨地域数据同步

### Q: 怎么支持高并发？
**A**: 详见 [高并发处理策略](./sections/04-高并发处理策略.md)

核心手段：
1. **流量控制**：限流、熔断、降级
2. **缓存前置**：多级缓存、热点数据预加载
3. **异步化**：MQ解耦、非核心链路异步处理
4. **弹性伸缩**：K8S HPA自动扩容
