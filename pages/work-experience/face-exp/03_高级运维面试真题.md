## 一、Linux 系统管理

### Q1: Linux 性能排查命令

**问题：** 服务器CPU使用率飙升，如何排查？

**标准答案：**

**排查流程：**

```bash
# 1. 查看整体CPU使用情况
top -c                    # 实时查看，显示完整命令
htop                      # 更友好的界面
mpstat -P ALL 1           # 查看各核心使用率

# 2. 找出高CPU进程
ps aux --sort=-%cpu | head -10    # CPU占用最高的10个进程
top -H -p <pid>                   # 查看进程的线程CPU使用

# 3. 分析进程详情
pidstat -p <pid> 1                # 进程资源使用详情
strace -p <pid>                   # 跟踪系统调用
lsof -p <pid>                     # 查看进程打开的文件

# 4. 查看系统调用
perf top -p <pid>                 # 实时查看热点函数
perf record -g -p <pid>           # 采样分析
perf report                       # 查看报告

# 5. 查看中断情况
cat /proc/interrupts              # 中断统计
mpstat -I SUM 1                   # 中断速率
```

**常见原因与解决：**

| 现象 | 可能原因 | 解决方案 |
|------|---------|---------|
| 用户态CPU高 | 应用计算密集 | 优化算法、增加资源 |
| 内核态CPU高 | 系统调用频繁 | 减少系统调用 |
| IO等待高 | 磁盘IO瓶颈 | 优化IO、使用SSD |
| 软中断高 | 网络包处理 | 优化网络配置 |

### Q2: 内存排查与优化

**问题：** 服务器内存不足，如何排查和优化？

**标准答案：**

```bash
# 1. 查看内存概况
free -h                           # 内存使用概况
cat /proc/meminfo                 # 详细内存信息
vmstat 1                          # 实时内存统计

# 2. 查看进程内存使用
ps aux --sort=-%mem | head -10    # 内存占用最高的进程
smem -t -k -s rss                 # 按RSS排序，显示总内存

# 3. 查看内存映射
pmap -x <pid>                     # 进程内存映射
cat /proc/<pid>/maps              # 内存映射详情
cat /proc/<pid>/smaps             # 带统计的内存映射

# 4. 分析内存泄漏
valgrind --leak-check=full ./app  # 内存泄漏检测
gdb -p <pid>                      # attach进程分析

# 5. 查看大页内存
cat /proc/meminfo | grep Huge     # 大页内存信息
```

**内存优化：**

```bash
# 调整vm参数
sysctl -w vm.swappiness=10        # 减少swap使用
sysctl -w vm.dirty_ratio=40       # 脏页比例
sysctl -w vm.dirty_background_ratio=10

# 调整透明大页
echo never > /sys/kernel/mm/transparent_hugepage/enabled

# 清理缓存
sync && echo 3 > /proc/sys/vm/drop_caches  # 清理缓存(慎用)
```

### Q3: 磁盘IO排查

**标准答案：**

```bash
# 1. 查看IO概况
iostat -x 1                       # 实时IO统计
iotop                             # 按进程查看IO

# 2. 查看磁盘状态
df -h                             # 磁盘使用情况
du -sh /*                         # 目录大小统计
fdisk -l                          # 磁盘分区
lsblk                             # 块设备信息

# 3. 查看IO调度
cat /sys/block/sda/queue/scheduler  # 当前调度算法
echo deadline > /sys/block/sda/queue/scheduler  # 修改调度算法

# 4. 磁盘性能测试
fio -name=test -filename=/dev/sda -bs=4k -size=1G \
    -rw=randread -iodepth=64 -numjobs=4 -time_based \
    -runtime=60 -group_reporting

# 5. 查看文件句柄
lsof | wc -l                      # 打开的文件总数
lsof -p <pid> | wc -l             # 进程打开的文件数
cat /proc/sys/fs/file-max         # 系统最大文件句柄
```

## 二、Docker 容器

### Q4: Docker 核心原理

**问题：** Docker是如何实现资源隔离的？

**标准答案：**

**三大核心技术：**

```
1. Namespace (命名空间)
   ├── PID: 进程隔离
   ├── NET: 网络隔离
   ├── IPC: 信号量隔离
   ├── MNT: 文件系统隔离
   ├── UTS: 主机名隔离
   └── USER: 用户隔离

2. Cgroups (控制组)
   ├── cpu: CPU资源限制
   ├── memory: 内存限制
   ├── blkio: IO限制
   └── devices: 设备访问控制

3. UnionFS (联合文件系统)
   └── 镜像分层存储
```

**Dockerfile最佳实践：**

```dockerfile
# 多阶段构建示例
# 阶段1: 构建
FROM golang:1.21-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -a -installsuffix cgo -o main .

# 阶段2: 运行
FROM alpine:latest
RUN apk --no-cache add ca-certificates
WORKDIR /root/
COPY --from=builder /app/main .
EXPOSE 8080
CMD ["./main"]
```

**资源限制：**

```bash
# 运行时资源限制
docker run -d \
    --name myapp \
    --cpus=2 \                    # CPU限制
    --memory=4g \                 # 内存限制
    --memory-swap=8g \            # 内存+Swap限制
    --pids-limit=1000 \           # 进程数限制
    --ulimit nofile=65535:65535 \ # 文件句柄限制
    myimage:latest
```

### Q8: K8s 调度原理

**标准答案：**

**调度流程：**

```
1. 预选 (Predicates)
   ├── PodFitsResources: 资源是否足够
   ├── PodFitsHostPorts: 端口是否冲突
   ├── PodMatchNodeSelector: 节点选择器匹配
   └── PodToleratesNodeTaints: 容忍度检查

2. 优选 (Priorities)
   ├── LeastRequestedPriority: 资源使用最少
   ├── BalancedResourceAllocation: 资源均衡
   ├── NodeAffinityPriority: 节点亲和性
   └── InterPodAffinityPriority: Pod亲和性

3. 绑定 (Bind)
   └── 将Pod绑定到选中的节点
```

**调度配置示例：**

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: myapp
spec:
  # 节点选择器
  nodeSelector:
    disktype: ssd
    
  # 节点亲和性
  affinity:
    nodeAffinity:
      requiredDuringSchedulingIgnoredDuringExecution:
        nodeSelectorTerms:
        - matchExpressions:
          - key: gpu
            operator: In
            values:
            - "true"
      preferredDuringSchedulingIgnoredDuringExecution:
      - weight: 100
        preference:
          matchExpressions:
          - key: zone
            operator: In
            values:
            - zone-a
            
  # 容忍度
  tolerations:
  - key: "dedicated"
    operator: "Equal"
    value: "gpu"
    effect: "NoSchedule"
    
  containers:
  - name: myapp
    image: myimage
    resources:
      requests:
        cpu: "500m"
        memory: "512Mi"
      limits:
        cpu: "2000m"
        memory: "2Gi"
```

```bash
# NVIDIA DCGM Exporter Dashboard ID: 12239
```

**GPU运维脚本：**

```bash
#!/bin/bash
# GPU监控告警脚本

THRESHOLD_UTIL=90    # GPU利用率阈值
THRESHOLD_MEM=90     # 显存使用阈值
THRESHOLD_TEMP=80    # 温度阈值

while true; do
    # 获取GPU信息
    gpu_info=$(nvidia-smi --query-gpu=index,utilization.gpu,memory.used,memory.total,temperature.gpu --format=csv,noheader,nounits)
    
    while IFS=',' read -r idx util mem_used mem_total temp; do
        # 计算显存使用率
        mem_util=$(echo "scale=0; $mem_used * 100 / $mem_total" | bc)
        
        # 检查告警条件
        if [ $util -gt $THRESHOLD_UTIL ]; then
            echo "[WARN] GPU $idx: 利用率 ${util}% > ${THRESHOLD_UTIL}%"
        fi
        
        if [ $mem_util -gt $THRESHOLD_MEM ]; then
            echo "[WARN] GPU $idx: 显存 ${mem_util}% > ${THRESHOLD_MEM}%"
        fi
        
        if [ $temp -gt $THRESHOLD_TEMP ]; then
            echo "[WARN] GPU $idx: 温度 ${temp}°C > ${THRESHOLD_TEMP}°C"
        fi
    done <<< "$gpu_info"
    
    sleep 60
done
```

```yaml
      - ./models:/models
      - ./logs:/logs
    ports:
      - "8000:8000"
    environment:
      - NCCL_P2P_DISABLE=1  # 多卡通信优化
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
    restart: unless-stopped
```

**vLLD 运维脚本：**

```bash
#!/bin/bash
# vLLM健康检查与自动重启

VLLM_URL="http://localhost:8000"
MAX_RETRIES=3
RETRY_COUNT=0

check_health() {
    response=$(curl -s -o /dev/null -w "%{http_code}" $VLLM_URL/health)
    if [ $response -eq 200 ]; then
        return 0
    else
        return 1
    fi
}

restart_vllm() {
    echo "[$(date)] 重启vLLM服务..."
    docker restart vllm-server
    sleep 60  # 等待服务启动
}

while true; do
    if ! check_health; then
        RETRY_COUNT=$((RETRY_COUNT + 1))
        echo "[WARN] vLLM健康检查失败 ($RETRY_COUNT/$MAX_RETRIES)"
        
        if [ $RETRY_COUNT -ge $MAX_RETRIES ]; then
            restart_vllm
            RETRY_COUNT=0
        fi
    else
        RETRY_COUNT=0
    fi
    
    sleep 30
done
```

**性能调优参数：**

| 参数 | 说明 | 建议值 |
|------|------|--------|
| tensor-parallel-size | 张量并行数 | GPU数量 |
| gpu-memory-utilization | 显存利用率 | 0.85-0.95 |
| max-model-len | 最大上下文长度 | 根据模型设置 |
| block-size | KV Cache块大小 | 16/32 |
| max-num-seqs | 最大并发序列数 | 256/512 |

**告警规则：**

```yaml
# rules/alerts.yml
groups:
  - name: node_alerts
    rules:
      # CPU使用率告警
      - alert: HighCPUUsage
        expr: 100 - (avg by(instance) (irate(node_cpu_seconds_total{mode="idle"}[5m])) * 100) > 80
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "CPU使用率过高"
          description: "实例 {{ $labels.instance }} CPU使用率 {{ $value }}%"
      
      # 内存使用率告警
      - alert: HighMemoryUsage
        expr: (1 - node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes) * 100 > 85
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "内存使用率过高"
          description: "实例 {{ $labels.instance }} 内存使用率 {{ $value }}%"
      
      # 磁盘使用率告警
      - alert: HighDiskUsage
        expr: (1 - node_filesystem_avail_bytes{fstype!="tmpfs"} / node_filesystem_size_bytes{fstype!="tmpfs"}) * 100 > 85
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "磁盘使用率过高"
          description: "实例 {{ $labels.instance }} 磁盘 {{ $labels.mountpoint }} 使用率 {{ $value }}%"

  - name: gpu_alerts
    rules:
      # GPU温度告警

        labels:
          severity: warning
        annotations:
          summary: "GPU温度过高"
          description: "GPU {{ $labels.gpu }} 温度 {{ $value }}°C"
      
      # GPU显存告警

        expr: DCGM_FI_DEV_MEM_COPY_UTIL > 90
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "GPU显存使用率过高"
```

### Q15: Ansible 自动化部署

**标准答案：**

**Ansible Playbook 示例：**

```yaml
# deploy-vllm.yml
- name: Deploy vLLM Server
  hosts: gpu_servers
  become: yes
  
  vars:
    model_path: /models/Qwen2.5-72B-Instruct
    gpu_count: 4
    
  tasks:
    # 检查GPU
    - name: Check GPU availability
      command: nvidia-smi -L
      register: gpu_check
      failed_when: gpu_check.rc != 0
      
    # 安装Docker
    - name: Install Docker
      apt:
        name: docker-ce
        state: present
        update_cache: yes
        
    # 拉取vLLM镜像
    - name: Pull vLLM image
      docker_image:
        name: vllm/vllm-openai
        tag: latest
        source: pull
        
    # 启动vLLM服务
    - name: Start vLLM container
      docker_container:
        name: vllm-server
        image: vllm/vllm-openai:latest
        state: started
        restart_policy: unless-stopped
        ports:
          - "8000:8000"
        volumes:
          - "{{ model_path }}:/models"
        command: >
          --model /models
          --tensor-parallel-size {{ gpu_count }}
          --gpu-memory-utilization 0.9
          --port 8000
        device_requests:
          - driver: nvidia
            count: all
            capabilities: [gpu]
            
    # 健康检查
    - name: Wait for vLLM to be ready
      uri:
        url: http://localhost:8000/health
        status_code: 200
      register: health_check
      until: health_check.status == 200
      retries: 30
      delay: 10
```
