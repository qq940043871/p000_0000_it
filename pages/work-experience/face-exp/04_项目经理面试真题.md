## 一、项目管理基础

### Q1: 项目管理五大过程组

**问题：** 请详细说明项目管理的五大过程组及其关键活动

**标准答案：**

```
┌─────────────────────────────────────────────────────────┐
│              项目管理五大过程组                          │
│                                                         │
│  1. 启动过程组 (Initiating)                             │
│     ├── 制定项目章程                                    │
│     ├── 识别干系人                                      │
│     └── 获得项目批准                                    │
│                                                         │
│  2. 规划过程组 (Planning)                               │
│     ├── 制定项目管理计划                                │
│     ├── 收集需求                                        │
│     ├── 定义范围 / WBS分解                              │
│     ├── 制定进度计划                                    │
│     ├── 估算成本 / 制定预算                              │
│     └── 规划风险应对                                    │
│                                                         │
│  3. 执行过程组 (Executing)                              │
│     ├── 指导与管理工作                                  │
│     ├── 管理质量                                        │
│     ├── 获取资源                                        │
│     ├── 建设团队                                        │
│     └── 管理干系人参与                                  │
│                                                         │
│  4. 监控过程组 (Monitoring & Controlling)               │
│     ├── 监控项目工作                                    │
│     ├── 实施整体变更控制                                │
│     ├── 确认范围                                        │
│     └── 控制风险                                        │
│                                                         │
│  5. 收尾过程组 (Closing)                                │
│     ├── 结束项目或阶段                                  │
│     └── 经验教训总结                                    │
└─────────────────────────────────────────────────────────┘
```

**AI项目特点：**

| 过程组 | AI项目特殊考虑 |
|--------|---------------|
| 启动 | 明确AI能力边界、设定可衡量目标 |
| 规划 | 预留模型调优时间、数据准备周期 |
| 执行 | 敏捷迭代、快速验证 |
| 监控 | 关注模型指标(准确率/延迟)、成本控制 |
| 收尾 | 模型文档化、部署运维交接 |

## 二、风险管理

### Q4: 风险识别与应对

**问题：** AI项目有哪些典型风险？如何应对？

**标准答案：**

**AI项目风险矩阵：**

| 风险类别 | 具体风险 | 概率 | 影响 | 应对策略 |
|---------|---------|------|------|---------|
| 技术风险 | 模型效果不达标 | 高 | 高 | 迭代验证、备选方案 |
| 技术风险 | 数据质量不足 | 中 | 高 | 数据治理、质量门禁 |
| 技术风险 | 性能不满足要求 | 中 | 中 | 优化迭代、架构调整 |
| 资源风险 | GPU资源不足 | 中 | 高 | 提前采购、云资源备用 |
| 资源风险 | 算法人才短缺 | 高 | 高 | 培训、外部合作 |
| 进度风险 | 需求变更频繁 | 高 | 中 | 敏捷迭代、变更控制 |
| 进度风险 | 依赖方延期 | 中 | 高 | 提前沟通、并行推进 |
| 成本风险 | 成本超支 | 中 | 高 | 预算控制、分阶段投入 |
| 合规风险 | 数据合规问题 | 低 | 高 | 合规审查、脱敏处理 |

**风险应对措施：**

```
1. 规避 (Avoid)
   例: 模型效果不确定 → 先做POC验证

2. 转移 (Transfer)
   例: 算法能力不足 → 外包或采购成熟方案

3. 减轻 (Mitigate)
   例: 数据质量风险 → 建立数据质量门禁

4. 接受 (Accept)
   例: 进度可能延期 → 预留缓冲时间
```

**风险监控表：**

```markdown
| 风险ID | 风险描述 | 责任人 | 状态 | 触发条件 | 应对行动 |
|--------|---------|--------|------|---------|---------|
| R001 | 模型准确率<80% | 张工 | 监控中 | 评估结果 | 调整模型/增加数据 |
| R002 | GPU资源不足 | 李工 | 已发生 | 使用率>90% | 申请扩容 |
| R003 | 需求变更 | 王经理 | 监控中 | 变更请求 | 评估影响/变更流程 |
```

## 三、AI 项目管理

### Q6: AI 项目生命周期

**问题：** AI项目与传统软件项目有何不同？如何管理？

**标准答案：**

**AI项目 vs 传统项目：**

| 维度 | 传统软件项目 | AI项目 |
|------|-------------|--------|
| 需求 | 明确、可量化 | 模糊、需探索 |
| 开发模式 | 瀑布/敏捷 | 实验驱动、迭代验证 |
| 成功标准 | 功能完整 | 模型指标达标 |
| 不确定性 | 较低 | 较高（模型效果不确定） |
| 技能要求 | 软件开发 | 算法+工程+业务 |
| 周期 | 可预估 | 难以精确预估 |

**AI项目生命周期：**

```
┌─────────────────────────────────────────────────────────┐
│                AI项目生命周期                            │
│                                                         │
│  Phase 1: 问题定义 (2-4周)                              │
│  ├── 业务问题理解                                       │
│  ├── AI可行性评估                                       │
│  ├── 成功指标定义                                       │
│  └── 项目范围确定                                       │
│                                                         │
│  Phase 2: 数据准备 (4-8周)                              │
│  ├── 数据收集                                           │
│  ├── 数据清洗                                           │
│  ├── 数据标注                                           │
│  └── 数据质量验证                                       │
│                                                         │
│  Phase 3: 模型开发 (4-12周)                             │
│  ├── 基线模型建立                                       │
│  ├── 模型迭代优化                                       │
│  ├── 模型评估验证                                       │
│  └── 模型选择决策                                       │
│                                                         │
│  Phase 4: 系统集成 (4-8周)                              │
│  ├── API开发                                            │
│  ├── 前端集成                                           │
│  ├── 性能优化                                           │
│  └── 系统测试                                           │
│                                                         │
│  Phase 5: 部署运维 (持续)                               │
│  ├── 生产部署                                           │

│  ├── 模型迭代                                           │

## 五、成本管理

### Q11: AI 项目成本估算

**标准答案：**

**成本构成：**

```
总成本 = 人力成本 + 硬件成本 + 软件成本 + 其他成本

1. 人力成本
   ├── 算法工程师: 2人 × 3月 × 40万/年 = 20万
   ├── 后端开发: 2人 × 2月 × 30万/年 = 10万
   ├── 前端开发: 1人 × 1月 × 25万/年 = 2万
   ├── 运维工程师: 1人 × 1月 × 25万/年 = 2万
   └── 项目经理: 1人 × 3月 × 35万/年 = 8.75万
   小计: 42.75万

2. 硬件成本
   ├── GPU服务器租赁: 4×A100 × 3月 × 2万/月 = 24万
   ├── 存储费用: 10TB × 3月 × 0.1万/月 = 3万
   └── 网络费用: 3月 × 0.5万/月 = 1.5万
   小计: 28.5万

3. 软件成本
   ├── 云服务费: 3月 × 1万/月 = 3万
   ├── 数据标注费: 10万条 × 0.5元/条 = 5万
   └── API调用费: 预估2万
   小计: 10万

4. 其他成本
   ├── 培训费用: 2万
   └── 预留风险金: 10%
   小计: 8万

总成本: 42.75 + 28.5 + 10 + 8 = 89.25万
```

**成本控制措施：**

| 措施 | 说明 |
|------|------|
| 分阶段投入 | 按里程碑分批投入，降低风险 |
| 资源共享 | 开发/测试环境资源共享 |
| 开源优先 | 优先使用开源组件降低成本 |
| 云资源按需 | 按需使用云资源，及时释放 |
| 成本监控 | 建立成本监控机制，及时预警 |

### Q12: ROI 分析

**标准答案：**

**ROI 计算示例：**

```
项目: 智能客服系统

- 运维成本: 10万/年

   - AI替代60%问题 → 减少12人
   - 节约人力成本: 180万/年

   - 响应时间: 5分钟 → 30秒
   - 客户满意度提升: 3.5 → 4.2
   - 转化率提升: 2% → 2.5%
   - 增收估算: 50万/年

3. 错误减少
   - 客服错误率: 5% → 1%
   - 减少投诉处理成本: 10万/年

年收益: 180 + 50 + 10 = 240万/年

ROI计算:
- 首年ROI = (240 - 80 - 10) / 90 × 100% = 167%
- 次年ROI = (240 - 10) / 10 × 100% = 2300%
- 3年累计ROI = (720 - 100) / 100 × 100% = 620%

```

3. 方案制定
   ├── 实施方案
   ├── 替代方案

4. 变更评审
   ├── CCB评审
   ├── 决策记录
   └── 通知相关方

5. 变更实施
   ├── 更新计划
   ├── 调整资源
   ├── 执行变更
   └── 验证效果
```
